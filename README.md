# Mind & Progress

아이디어를 구조화하는 마인드맵과 실제 업무 진행률을 한 화면에서 연결하는 웹 서비스의 초기 프로토타입입니다.

## 실행

```bash
npm install
npm run dev
```

최초 실행 시 관리자 계정이 없으면 API 서버 콘솔에 일회성 임시 비밀번호가 표시됩니다. 초기 계정을 직접 지정하려면 실행 전에 환경변수를 설정합니다.

```powershell
$env:MNP_ADMIN_EMAIL='admin@example.com'
$env:MNP_ADMIN_PASSWORD='8자 이상의 안전한 비밀번호'
npm run dev
```

환경변수는 관리자 계정을 처음 생성할 때만 적용됩니다. 로그인 후 사용자 메뉴의 `비밀번호 변경`에서 비밀번호를 교체해 주세요. 문서, 댓글, 계정 및 변경 이력이 저장되는 `server/data`의 런타임 파일은 Git에 포함되지 않습니다.

## 현재 구현 범위

- 노드 선택, 이동, 연결, 추가 및 삭제
- 노드별 제목, 설명, 상태, 진행률 편집
- 서버 세션 로그인과 편집자/뷰어 권한 검증
- 뷰어의 저장 요청 서버 차단
- 여러 마인드맵 문서 생성, 전환, 이름 변경
- 문서별 서버 자동 저장과 로컬 백업
- 노드를 업무로 전환하고 담당자·마감일 지정
- 체크리스트 완료율과 노드 진행률 자동 동기화
- 칸반에서 업무 상태별 조회와 드래그 이동
- 마감일 기준 업무 타임라인
- 완료율, 기한 초과, 담당자 분배 진행률 대시보드
- 문서별 최대 100단계 실행 취소·다시 실행 (`Ctrl+Z`, `Ctrl+Y`)
- 미니맵, 확대/축소, 전체 보기
- 하위 가지 접기·전체 펼치기와 노드 검색·상태·담당자 필터
- 업무 링크 연결 및 완료 상태 표시
- 문서별 최근 50개 변경 이력 조회와 안전한 이전 버전 복원
- 다른 브라우저의 문서 변경 실시간 감지와 뷰어 자동 반영
- 같은 문서 접속자와 실시간 캔버스 커서 표시
- 문서 버전 충돌 감지와 노드·필드 단위 3방향 자동 병합
- 노드별 실시간 댓글, 답글 스레드, 멘션, 해결 상태와 이모지 반응
- 사용자별 실시간 알림함과 개별·전체 읽음 처리
- 로그인 계정 담당자 배정과 마감 3일 전·당일·기한 초과 개인 알림
- 관리자 전용 편집자 계정 생성·수정·정지·비밀번호 초기화·삭제
- 로그인 사용자의 현재 비밀번호 확인 기반 비밀번호 변경
- 선행·후속 업무 관계, 순환 방지와 차단 병목 표시

## 접속 경로

- 일반 로그인: `http://127.0.0.1:4175/`
- 공유 화면: `http://127.0.0.1:4175/{탭}/{문서 ID}/{노드 ID}`

공유 화면은 먼저 현재 로그인 세션을 확인해 로그인 계정 권한으로 동작하고, 로그인하지 않은 경우에만 공개 읽기 전용 계정으로 연결합니다. 탭에는 `마인드맵`, `칸반`, `타임라인`, `대시보드` 또는 각각의 영문 식별자인 `mindmap`, `kanban`, `timeline`, `dashboard`를 사용할 수 있습니다. 노드가 없으면 문서만 열고, 문서가 없으면 해당 탭의 첫 문서를 엽니다. 칸반·타임라인·대시보드에서는 업무 관리가 활성화된 노드만 선택됩니다.

로그인 화면은 계정 정보를 미리 채우지 않습니다. 로그인에 성공한 이메일만 해당 브라우저에 기억하며 비밀번호는 저장하지 않습니다. `로그인 유지`를 선택하지 않으면 최대 8시간의 브라우저 세션을 사용하고, 선택하면 토큰 원문을 저장하지 않는 장기 세션으로 서버 재시작 후에도 최대 30일간 로그인을 유지합니다.

공개 읽기 전용 화면은 전용 시스템 계정으로 분리되어 문서와 댓글 조회 및 업무 링크 열기만 허용합니다. 댓글 작성·반응과 문서 변경은 서버에서도 차단합니다. 편집자는 로그인 후 우측 상단 사용자 이름을 눌러 `비밀번호 변경`을 선택할 수 있습니다.

## AionUi 연동

편집자가 카드를 선택하고 `AI 대화 시작`을 누르면 실행 중인 AionUi에서 AI 종류, 모델, 권한, 사고 수준, 스킬, MCP와 작업공간 옵션을 가져옵니다. 첫 메시지에는 문서 ID와 선택 카드 ID, 서버가 검증한 작성자 귀속 토큰을 전달합니다. AI는 `mindnprogress_get_context`를 호출해 제품 개념과 작성 규칙, 최신 문서 구조와 선택 카드 정보를 함께 확인합니다. 이후 댓글과 변경 이력에는 작성자가 `{AI 종류}({모델})` 형식으로 기록됩니다. 선택 카드나 최상위 카드에 업무 링크가 있으면 작업 전에 링크의 업무 본문, 댓글, 첨부파일 목록과 관련 링크를 조사하되 특정 자료가 있다고 가정하지 않습니다. 응답은 AionUi에서만 처리합니다.

AionUi 데스크톱 앱은 실행할 때 결정된 백엔드 포트를 운영체제 임시 디렉터리의 `aionui-backend.json`에 게시합니다. MindNProgress는 AI 옵션이나 대화를 시작할 때마다 이 정보를 다시 읽으므로 AionUi를 재실행해 포트가 바뀌어도 자동으로 다시 연결합니다. 게시 파일이 없거나 오래된 버전이면 `http://127.0.0.1:1986`, `http://127.0.0.1:5830` 순서로 호환 연결을 시도합니다. 다른 주소를 명시하려면 MindNProgress API 서버를 실행하기 전에 `MNP_AIONUI_URL` 환경변수로 지정합니다. AionUi에는 다음 로컬 MCP 서버를 등록하고 활성화해야 합니다.

```text
이름: MindNProgress
전송 방식: stdio
명령: node
인수: C:\Git\MindNProgress\mcp\server.mjs
```

MCP는 `server/data/_integration-token`의 자동 생성 로컬 토큰으로 인증합니다. 이 파일은 Git에 포함되지 않습니다. 문서, 카드, 업무, 체크리스트, 관계, 댓글, 이력, 휴지통과 알림 도구를 제공하며 비밀번호 변경 도구는 제공하지 않습니다. 휴지통 문서는 복원하거나 선택 영구 삭제할 수 있고, 전체 비우기는 별도의 영구 삭제 확인값이 있어야 실행됩니다. MindNProgress 밖에서 시작한 AI는 문서 ID 없이 `mindnprogress_read_me_first`를 호출해 제품 개념과 권장 작업 순서를 확인할 수 있습니다. 여러 카드로 구성된 새 문서는 `mindnprogress_create_mindmap`으로 한 번에 생성해 중간 저장의 버전 충돌을 방지합니다.

MCP 전체 회귀 검증은 운영 데이터와 분리된 임시 API 서버에서 실행됩니다. 등록된 모든 도구의 성공 경로와 버전 충돌·순환 이동·루트 삭제·영구 삭제 확인 같은 안전 경계를 함께 검사하며 테스트 데이터는 종료 시 제거합니다.

```bash
npm run test:mcp
```
